Protocolo HTTP

HTTP vem de Hypertext Transfer Protocol(Protocolo de Transferência de Hipertexto). É um protocolo, uma forma de conversa entre duas máquinas, que permite transferir um hipertexto de um lado a outro.

Cliente e Servidor

O cliente e servidor se comunicam através de trocas de mensagens. As mensagens enviadas pelo cliente, geralmente um navegador da Web, são chamadas de requisições ou solicitações(requests), as mensagens que são enviadas pelo servidor como resposta são chamadas de respostas(responses).

No exemplo acima, eu simplesmente recarreguei a página, dessa forma vemos o navegador pedindo recursos para o servidor daquele site que recarregamos, ele está fazendo essa requisição justamente para poder recarregar a página e gerar o resultado para o usuário. No caso da response é exatamente a resposta que ele dá para o usuário, podemos ver na imagem que em status, todos deram 200, o que isso significa? Siginifica que está tudo OK!

Na request para ela pedir dados para o servidor, ela utiliza alguns métodos, entre eles pode encontrar alguns desses abaixo:

  • GET:Solicita dados do servidor.
  • POST:Envia dados para o servidor.
  • DELETE:Remove dados para o servidor.
